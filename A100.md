# A100 部署 / 压测说明

这版在原 CPU 工程上加了 **GPU 吞吐路径**，模型结构不变，可直接上多卡 A100。

## 新增了什么

| 文件 | 作用 |
|---|---|
| `dlo_wm/data/batch.py` | **disjoint 批图**：把 B 条绳拼成一张大图，GPU 吞吐的前提 |
| `dlo_wm/model/gnn_batched.py` | 批图版 GNN，一次前向处理整个 batch，图级 head 用 segment pooling |
| `scripts/benchmark.py` | **单卡压测**：逐档扫 batch/节点/模型规模，找 80GB 的量级上限 |
| `scripts/train_ddp.py` | 多卡 DDP + AMP(bf16) 训练 |
| `configs/a100.py` | A100 大规模起点超参 |

## 第一步：压测，吸到什么量级（你这轮的目标）

在一张 A100 上先跑单卡压测：

```bash
python scripts/benchmark.py
```

它逐档加压（batch 256→1024、节点 64→128、hidden 256→512、mp 10→12），
每档打印 **samples/s + 显存峰值**，OOM 的档标 `OOM`。
找到最大的 `OK` 档作为单卡设置。**DDP 多卡总吞吐 ≈ 单卡 × 卡数**（数据并行）。

> 沙盒无 GPU 跑不了这个脚本，但代码已在 CPU 上验证逻辑正确、可导入。
> 上 A100 直接跑即可。

## 第二步：多卡训练

```bash
torchrun --standalone --nproc_per_node=<卡数> scripts/train_ddp.py
```

- 每卡一份模型，梯度自动 all-reduce
- AMP 用 **bf16**（A100 原生，比 fp16 数值稳，不需要 GradScaler）
- 当前每卡用不同 seed 生成合成轨迹 ≈ 数据并行

## 调优要点（A100 80GB）

1. **先把单卡吃满再上多卡**。GNN 在 GPU 上利用率低往往是 batch 太小，
   先用 benchmark 把 batch 推到显存上限附近。
2. **bf16 已默认开**。若想再省显存上更大 batch，可加 activation checkpointing
   （message passing 层数多时收益大）。
3. **节点数 vs batch 的权衡**：真实绳子 64~128 节点够用；
   节点翻倍 ≈ 边翻倍 ≈ 显存和算力都涨，压测表会告诉你怎么换。
4. **数据喂养**：当前合成数据在主进程造，量级大时会成瓶颈。
   接真实仿真数据时，用 DataLoader 多 worker + DistributedSampler
   预取，别让 GPU 等 CPU。

## 接真实数据时改两处
- `scripts/train_ddp.py` 里的 per-rank `SyntheticRope` → 你的 `TrajectoryProvider`
  + 真正的 `DistributedSampler` 切分数据集
- 其余（批图、模型、损失、DDP）不用动

## 已知数值/工程注意
- 长 horizon rollout 误差累积仍在 —— pushforward/noise injection 在单图 trainer 里，
  批图版要把它加到 `train_ddp.py`（按需）
- 拓扑变化瞬间的接触边重连，复杂任务建议混合几何/符号判定兜底（同主 README）
